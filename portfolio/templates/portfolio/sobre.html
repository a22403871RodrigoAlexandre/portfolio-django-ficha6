Arquitetura MVT

Model

Camada de dados

Define a estrutura da base de dados. Cada classe em models.py corresponde a uma tabela. O Django gere automaticamente as queries SQL através do ORM.

View

Camada de lógica

Funções em views.py que recebem um pedido HTTP, recolhem dados do Model e devolvem uma resposta — normalmente um template HTML renderizado.

Template

Camada de apresentação

Ficheiros HTML que usam a linguagem de templates do Django ({{ }} e {% %}) para inserir dinamicamente os dados recebidos da View.

Fluxo de Informação

Como funciona um pedido

  • 1. Browser faz um pedido HTTP
  • 2. urls.py encaminha para a View certa
  • 3. View consulta o Model (base de dados)
  • 4. View passa os dados ao Template
  • 5. Template renderiza o HTML
  • 6. HTML é devolvido ao browser

models.py

Definição dos modelos Django

class Licenciatura(models.Model):
    nome, sigla, instituicao, descricao
    ano_inicio, ano_fim, url_curso

class Docente(models.Model):
    nome, email, url_pagina_lusofona
    linkedin, foto

class UnidadeCurricular(models.Model):
    nome, sigla, ano_curricular, semestre, ects
    licenciatura (FK) → Licenciatura
    docentes (M2M) → Docente

class Projeto(models.Model):
    titulo, descricao, url_github
    uc (FK) → UnidadeCurricular
    tecnologias (M2M) → Tecnologia

class Tecnologia(models.Model):
    nome, descricao, categoria, logo, url

class TFC(models.Model):
    titulo, autor, resumo, rating
    licenciatura (FK) → Licenciatura
    orientador (M2M) → Docente

class Competencia(models.Model):
    nome, tipo, nivel, descricao
    tecnologias (M2M) → Tecnologia

class Formacao(models.Model):
    titulo, instituicao, tipo
    tecnologias (M2M) → Tecnologia
